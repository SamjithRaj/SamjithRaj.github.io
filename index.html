A bit about me

I enjoy working close to the fundamentals — memory models, synchronization primitives, and performance optimization. I prefer systems that fail loudly instead of silently.

My focus is backend engineering: building components that are predictable under load, easy to reason about, and correct by design. I gravitate toward low-level abstractions because they reveal what the machine is actually doing.

Outside of code I think about distributed systems tradeoffs, latency budgets, and writing software that future maintainers will genuinely appreciate.

02 — Projects

Things I've built

Mini Redis — In-Memory Key-Value Store

A Redis-like database engine in pure C++. Supports O(1) key retrieval via unordered maps, mutex-based concurrency for safe multi-threaded access, and persistent snapshot storage. Command handler and database layers cleanly separated.

C++ Concurrency STL Snapshots

LRU Cache — Constant Time Eviction

O(1) LRU cache using a hash map paired with a doubly linked list. Extended with thread-safe access via fine-grained locking — designed to handle concurrent reads and writes without becoming a bottleneck.

C++ Data Structures Thread-Safe

The Giving Plate — Hack4SDG

Hackathon project tackling UN SDG Zero Hunger. A full-stack Flask platform with Firebase auth and an AI matching algorithm that connects food donors with nearby community recipients. 3-person team, HTML/CSS/Python/PHP stack.
